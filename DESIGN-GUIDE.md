## 1. ページの型（章構成）

領域Questページは以下の順で構成する:

1. **Hero** — 誰向けの何のサイトかを一発で伝える＋要点3つ（AIO）
2. **01 WHY NOW** — なぜ今この領域か。数字バンド＋カード6枚＋出典
3. **02 JOB MAP** — 職種マップ8職種（必要スキル付き）〔白帯〕
4. **03 SKILL BRIDGE** — 異業種経験の活かし方8パターン
5. **04 SELF CHECK** — 蛇行タイムライン＋診断導線〔白帯〕
6. **05 PLAYER MAP** — 企業・機関マップ（公式リンク付き）
7. **06 BLOG & CONTENTS** — note最新3件＋コンテンツ導線〔白帯〕
8. **07 FAQ** — 3問（AIO・FAQPage JSON-LD）
9. **遊び心セクション** — 実寸比チャート等。番号なし・終盤に置く
10. **CTA** — 個人相談＋企業相談の2枚看板

実装は `app/robotics/page.tsx` を正とする（SectionHead・QuestFx・quest-* CSSを流用）。

## 2. 見出しシステム（45+型）

- 章番号: `<span class="qno">01</span>`（丸囲い）＋英字キッカー（letter-spacing広め）
- 背景に大きく薄いゴーストテキスト（`.quest-ghost`）。**ゴーストは日本語の項目タイトルにする**（ユーザーFB・2026-07-09）:
  - キッカーは英字のまま、その後ろの大きなゴーストで「この章が何の話か」を日本語で伝える（企業側・求職者側どちらのユーザーにも一目で伝わるように）
  - フォントは `--font-serif-ja`（明朝）・基準サイズ `clamp(40px, 7.5vw, 84px)`。長文でもはみ出さないよう文字数に応じた上限を `ghostFontSize()`（app/robotics/page.tsx）でかける
  - /roboticsの文言（ミネさん確定・2026-07-09）: Why Now=「なぜ、今、水中ロボティクスが重要なのか」/ Job Map=「水中ロボティクスの仕事」/ Skill Bridge=「こんな経験が活きる」/ Self Check=「キャリア診断」/ Player Map=「企業と研究機関」/ Blog & Contents=「ブログ」/ FAQ=「よくある質問」/ Deep Dive=「どこまで潜れるのか」
  - 横展開時も同じ型で: 英字キッカー＋日本語ゴースト（英字ゴーストの再発明禁止）。文言は説明的でよい（短さより「何の話か伝わること」を優先）
- h2は「主張」にする（説明文にしない）。例:「国が、2030年までのAUV産業化を宣言している。」
- h2の強調は `<em>` ＝ 色変え＋**オレンジのマーカー風下線**（`linear-gradient(transparent 68%, rgba(255,122,33,.5) 68%)`）
- 写真背景上の章は白文字系＋text-shadow、白帯上の章はインク色（`.quest-sec:not(.quest-sec-alt)` の切替が実装済み）

## 3. 余白リズム（ユーザーFBで確立・2026-07-08）

- `.section` 既定（上96px/下0）は使わない。**questページは上下対称**:
  - 通常セクション: `padding: 84px 24px`
  - 白帯セクション: `padding: 76px 48px` ＋ **前後に margin 40px**
  - 最終CTA: 下110px（フッター前の余白）
  - モバイル(≤720px): 56px / 48px+24px / 80px
- セクション同士が密着して見えたら余白の非対称を疑う（過去の不具合原因）

## 4. 背景・写真（ユーザーFBで確立・2026-07-08）

- **ページ背景の写真にオーバーレイ（薄い膜）をかけない。写真そのまま**。可読性は白カード・text-shadow・白帯で確保する
- **/roboticsの背景 = 深度ジャーニー**（`components/DepthJourney.tsx`・ユーザーFB・2026-07-12）: スクロールに連動して「水面直下 → 気泡の浅層 → クラゲの中深層 → 発光クラゲの深海 → 海底（沈没船 deep-dive.jpg）」をクロスフェード。道中の4枚はミネさん支給（journey-*.jpg・Unsplash）で、各層に可読性の暗色グラデを焼き込む（明るい層ほど濃く）
  - 右下に深度計（スクロール量→水深換算・最大8,000m=うらしま8000。浅い層はゆっくり潜るpowカーブ）。遊び心＋潜行感の核
  - 明るい表層レイヤーには他サブページ同様の暗色グラデを焼き込む（白文字の可読性優先。「オーバーレイ禁止」ルールの例外）
  - **02 Job Map / 03 Skill Bridge / 05 Player Mapは白帯・白カードを使わない**（ユーザーFB・2026-07-12）。海が透けるダークガラスパネル（`rgba(2,26,36,0.55)`+ティール枠+blur）で「道中の海」を見せる。白帯が残るのは04 Self Checkと06 Blogのみ
- 白帯（`.quest-sec-alt`）は `rgba(251,255,253,0.82)`＋角丸24px。写真背景と交互に置いてリズムを作る
- 素材は権利がクリアなもののみ（Unsplash可）。政府資料内の写真（©JAMSTEC等）は転載不可

## 5. ヒーロー原則

- **オリジナルのアニメーションSVGシーン**を描き起こす（写真がなくても成立し、他サイトと被らない）
- ロード時から動く: /roboticsはAUV浮遊＋プロペラ回転＋ソナーリング＋ビーコン明滅＋ライトビーム＋海底スキャン＋気泡＋マリンスノー＋光の差し込み
- **ページの主張を背負う小物**を必ず1つ入れる（/robotics = 右端の深度メーター「-40m ダイバーの限界 / -200m AUVの現場」）
- 構図: テキスト左・シーン右。要点3つはガラスパネル（`.qhv-points`）
- **h1は「知らなかった」ギャップのある相対比較の事実で引き込む**（ユーザーFB・2026-07-09）。/robotics=「人が潜れるのは、40m。日本の水中ロボットは、8,000m。」（呼称は「水中ロボット」で統一。ROV/AUVはh1では使わない・「潜水艦」は有人軍用艦を指すので不可）。機能説明のh1（「◯◯を仕事にする」等）より、読者が読み進めたくなる事実を置く。何のサイトかはバッジとリード末尾（「〜のためのキャリアサイトです」）で回収する
  - **数字のすごさは「物差し」とセットで示す**（ユーザーFB・2026-07-09）。8,000mだけでは凄さが伝わらないので、上限（最深部チャレンジャー海淵 約10,920m）を添える。届いていない差分は欠点ではなく「これから入る人の伸び代」として語る
- **キャッチフレーズは領域固有の事実を使い、汎用の強ネタは取り合わない**（ユーザーFB・2026-07-09）。「国土は世界61位。海は世界6位。」（EEZ）は海洋産業全体の共有資産なので、親和性が最も高い領域（海底資源・海洋防衛・親サイトトップ等）のために温存する。各Questのh1は、その領域でしか言えない相対比較を探すこと
- 下部にSCROLLキュー。`prefers-reduced-motion` で全アニメ停止。モバイルはシーン減光＋メーター非表示
- **領域プレート**（`.quest-plate`・ユーザーFB 2026-07-12: 旧ピルバッジは小さすぎた）: 「OCEAN QUEST FAMILY」小+「QUEST <連番>」オレンジ+「◯◯のしごと」大(27px)の3段ロックアップ。オレンジの左ボーダー+ガラスパネル。h1より一段小さい主役級の看板にする
- **領域名は「QUEST <連番> ◯◯のしごと」型**（ミネさん確定・2026-07-12。例: QUEST 01 水中ロボティクスのしごと）。`lib/content.ts` の `questName` が単一ソースで、ヘッダー/フッター/診断結果に自動反映。新領域は連番を進めて追加する

## 6. コンテンツ原則

- **WHY NOWは政府一次資料ベース**。ローカルの `新規事業調査/海洋産業/政府公開海洋産業/` から数字を取る
  - 数字バンド（`.quest-stats-band`）: ダークタイル4枚・大きな数字はオレンジ・カウントアップ演出
  - 出典を必ず明記し、試算値には「一定の条件設定に基づく目安」の注記を付ける
  - カードは**数字リード型6枚**（`.quest-why-num`。ユーザーFB・2026-07-09: 文章カードは読まれない。大きな数字→主張→2〜3文の説明の順）
  - カードにはタグを付け、**「日本の必然 → いま起きている変化 → 産業化後の世界」の物語順**で2枚ずつ並べる。数字と「なぜ今か」の紐づきをタグで明示する
  - 「なぜこの技術/産業でないといけないのか」の必然性を最低2枚（タグ「日本の必然」）、**産業化・普及後にどういう世界線が待つのか**を最低1枚入れる
- **Skill Bridgeは文章の行列にしない**（ユーザーFB・2026-07-12）。`.quest-bridge-card`=「いまの仕事(白太字)→潜る点線+オレンジ矢頭→海でのしごと(ティール太字)」の縦ブリッジカード2列。タグ(いまの仕事/海でのしごと)でスキャン可能にし、補足は小さな1文
- **語彙**: 「翻訳できる」は使わない →「あなたの経験は、◯◯で武器になる。」（Skill Bridge）。「活かせる」も面白みがないので避ける
- **呼称は「水中ロボット」で統一**（ユーザーFB・2026-07-09）。単独の「ロボット」「海洋ロボット」は使わない（異業種側の名称「FA・産業用ロボット」等は除く）。「潜水艦」は有人軍用艦を指すので不可
- **専門用語には入門パネルを添える**（ユーザーFB・2026-07-09）。/robotics=WHY NOW内の「そもそも、ROV・AUVとは？」（`.quest-rovauv`。イラスト対比: ROV=船+ケーブル+操縦 / AUV=単独航行+ソナー）。初見の求職者が置いていかれる略語をページ前半で回収する
- **ロードマップはカード列ではなくタイムラインUI**（ユーザーFB・2026-07-09）。`.quest-roadmap`=グラデーション線（ティール→オレンジ）+ノード。最終ステップ（目指す世界）だけオレンジで強調。モバイルは縦タイムライン
- **Job Mapカードはアイコンではなく実際の作業風景の写真**（ユーザーFB・2026-07-09）。lucideアイコンはAI感が出るので職種カードでは使わない。写真はUnsplash等権利クリアのみ・出典IDを `public/images/roles/README.md` に記録
- **企業名を出すなら公式サイトへのリンクチップ必須**（`.quest-player-links`・別タブ）。「代表例で網羅ではない」と明記
- **遊び心セクション**: 実寸比のチャートで驚きを作る（/robotics = 潜航深度比較: ダイバー40m→逆さ富士→しんかい6500→うらしま8000→逆さエベレスト→悟空号（中国AUV・10,896m）→チャレンジャー海淵10,920m）。自虐的な注記で笑わせて締める（「ダイバーのバーが見えないのは間違いではありません」）
  - **チャートはバーではなくイラストで見せる**（ユーザーFB・2026-07-09）: 人=泳ぐダイバーのシルエット、山=雪冠付きの逆さシルエット、機体=点線の降下ライン+機体イラスト（`.qdive-vehicle`）
  - **同種のモチーフでもシルエットを描き分ける**（ユーザーFB・2026-07-09）: 富士山=なだらかな円錐+雪冠 / エベレスト=雪の山塊+暗い岩の主峰。**チャレンジャー海淵は深海グラデーションの柱のまま**（海溝断面図の案はユーザーFBで不採用・2026-07-10）
  - **最深の基準（海淵）より機体が深く見えないように**: 機体イラストの下端=降下ラインの終点（最深到達点）に揃える（下へはみ出させない）
  - **機体名は固有名詞+（分類）**: うらしま8000（AUV・日本）/ 悟空号（AUV・中国）/ しんかい6500（有人潜水調査船・日本）。**しんかい6500を「ROV」と表記しない**（人が乗る機体。ROVは「かいこう」）
  - 日中のAUV競争（うらしま8000 vs 悟空号10,896m世界記録）を「WHY NOWの必然」として本文でも回収する
- 英単語の混入禁止（過去に「widen」混入事故）。日本語で書き切る

## 8.5 OGP（SNS/Slackリンクプレビュー）

- **OGP画像は写真そのままにしない**。「何のサイトか」が一目で伝わるブランド合成画像にする（ユーザーFB・2026-07-09）
- **ページごとに個別OGP**（ユーザーFB・2026-07-09）。実装の正本は `lib/og/render.tsx`（全ページ共通レンダラー）:
  - 共通フレーム = ロゴ（左上）＋ドメイン＋「運営 POTENTIALIGHT」（下段）＋背景写真＋左からの可読性グラデーション。**この骨格は全ページで崩さない**（一目でOcean Questと分かる一貫性がクリック率の土台）
  - 中央モチーフだけをページの役割で切替: standard（トップ・企業・診断・地図・相談・Quest）/ video（再生ボタン+シークバー）/ article（白カード+タイトル）/ ebook（表紙スタック+無料バッジ）/ event（チケット風破線バナー）/ interview（人物円形写真+引用・**将来用**）/ news（マストヘッド+見出し帯・**将来用**）
  - ページ追加時: `lib/og/render.tsx` の `ogPages` に設定を足し、`app/<route>/opengraph-image.tsx` と `twitter-image.tsx`（各10行のラッパー）を置く
  - 明るい背景写真のページは `shade: "strong"` でグラデーションを濃くする
- **og:title/og:descriptionもページ個別必須**。Next.jsのメタデータは浅いマージのため、ページ側は必ず `lib/seo.ts` の `pageMetadata()` を使う（使わないとOGタイトルがサイト共通のままになる）
- **差し替え規約**: `public/images/ogp/<ページキー>.png`（1200×630）を置くと生成画像より優先して配信される（デザイナー入稿・AI生成画像用。手順とAIプロンプトは同フォルダのREADME.md）
- 重要要素は中央1200×630の内側1000×524に収める（LINE/Discordの端切れ対策）。タイトルは最大2行・1行13文字前後
- フォントは `assets/fonts/ZenKakuGothicNew-*.ttf`（eBookカバーと共通）
- satoriの制約: `style` にundefined値を渡すと落ちる。条件付きスタイルはスプレッド `...(cond ? {...} : {})` で書く
- 注意: SlackはOGPをキャッシュする。更新確認は新しいチャンネル/メッセージで貼り直すか、Slackの場合はURL末尾に `?v=2` 等を付けて確認
